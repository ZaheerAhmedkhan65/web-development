JavaScript Object-Oriented Programming (OOP) Lecture Notes

1. Introduction to Object-Oriented Programming (OOP)

OOP is a programming paradigm based on the concept of "objects". Objects are instances of classes and are used to model real-world entities. The main concepts of OOP are:

  • Class: A blueprint for creating objects.
  • Object: An instance of a class.
  • Constructor: A special method used for initializing objects.
  • Encapsulation: The bundling of data (properties) and methods (functions) into a single unit (class/object).
  • Inheritance: The mechanism by which one class can inherit properties and methods from another class.
  • Polymorphism: The ability for different classes to be treated as instances of the same class through inheritance.
  • Abstraction: Hiding the complex implementation details and exposing only the necessary parts of an object.

2. Creating Objects in JavaScript

In JavaScript, you can create objects in multiple ways:

Using Object Literals:


const car = {
  make: 'Toyota',
  model: 'Corolla',
  start: function() {
    console.log('Car started');
  }
};

console.log(car.make); // Output: Toyota
car.start();           // Output: Car started

3. JavaScript Classes

A class in JavaScript is a blueprint for creating objects. It defines properties and methods that objects created from the class will have.

Example of a JavaScript Class:


class Car {
  constructor(make, model, year) {
    this.make = make;
    this.model = model;
    this.year = year;
  }

  start() {
    console.log(`${this.make} ${this.model} started`);
  }

  displayInfo() {
    console.log(`Car: ${this.make}, Model: ${this.model}, Year: ${this.year}`);
  }
}

// Create an instance of Car
const myCar = new Car('Honda', 'Civic', 2020);
myCar.start();         // Output: Honda Civic started
myCar.displayInfo();   // Output: Car: Honda, Model: Civic, Year: 2020

4. Encapsulation

Encapsulation means keeping properties and methods private or protected, allowing access to them through public methods only.

Private Variables Using # Symbol:


class BankAccount {
  #balance = 0; // private variable

  constructor(owner, balance) {
    this.owner = owner;
    this.#balance = balance;
  }

  deposit(amount) {
    this.#balance += amount;
    console.log(`Deposited: $${amount}`);
  }

  getBalance() {
    return this.#balance;
  }
}

const myAccount = new BankAccount('John', 500);
myAccount.deposit(100);        // Output: Deposited: $100
console.log(myAccount.getBalance());  // Output: 600

5. Inheritance

Inheritance allows one class to extend another, inheriting its properties and methods.

Example of Inheritance:


class Vehicle {
  constructor(type, make, model) {
    this.type = type;
    this.make = make;
    this.model = model;
  }

  start() {
    console.log(`${this.type} ${this.make} ${this.model} started`);
  }
}

class Car extends Vehicle {
  constructor(make, model, year) {
    super('Car', make, model); // Call the parent class constructor
    this.year = year;
  }

  displayCarInfo() {
    console.log(`Car Info: ${this.make}, Model: ${this.model}, Year: ${this.year}`);
  }
}

const myCar = new Car('Tesla', 'Model S', 2021);
myCar.start();               // Output: Car Tesla Model S started
myCar.displayCarInfo();       // Output: Car Info: Tesla, Model: Model S, Year: 2021

6. Polymorphism

Polymorphism allows methods to be overridden, providing different functionality in the child class while keeping the same method name.

Method Overriding:


class Animal {
  speak() {
    console.log('Animal makes a sound');
  }
}

class Dog extends Animal {
  speak() {
    console.log('Dog barks');
  }
}

const myDog = new Dog();
myDog.speak(); // Output: Dog barks

7. Abstraction

Abstraction means hiding the internal details of the object and showing only the functionality. In JavaScript, there are no strict abstract classes, but you can simulate abstraction using methods and encapsulation.

8. Conclusion

OOP in JavaScript allows you to create reusable, maintainable, and scalable code by utilizing the concepts of classes, objects, inheritance, and more. By modeling real-world entities with OOP, you can create complex applications with clean and understandable code.
