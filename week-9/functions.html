JavaScript Functions - Lecture Notes

1. Introduction to Functions

Definition: A function is a block of reusable code that performs a specific task.

Purpose: Functions help make your code modular, organized, and easier to maintain.

Syntax:

function functionName(parameters) {
    // code to be executed
}

2. Why Use Functions?

  • Reusability: Write a piece of code once and use it multiple times.
  • Modularity: Break your code into manageable chunks.
  • Readability: Improves the structure and readability of the code.

3. Function Declaration

Basic Syntax:

function greet() {
    console.log("Hello, World!");
}

Invocation: Call a function using its name followed by parentheses.

greet();  // Outputs: Hello, World!

4. Function Parameters and Arguments

Parameters: Variables listed as part of the function definition.

Arguments: Values passed to the function when it is called.

Example:

function greet(name) {
    console.log("Hello, " + name + "!");
}
greet("Alice");  // Outputs: Hello, Alice!

5. Return Statement

Purpose: Returns a value from the function and stops the execution of the function.

Syntax:

function add(a, b) {
    return a + b;
}
let result = add(5, 3);  // result is 8

Important: Without a return statement, a function returns undefined.

6. Function Expressions

Definition: A function can also be defined as an expression and stored in a variable.

Syntax:

let sum = function(a, b) {
    return a + b;
};
console.log(sum(5, 3));  // Outputs: 8

Anonymous Functions: Functions without a name.

Use Cases: Function expressions are useful for passing functions as arguments or using them as callbacks.

7. Arrow Functions

Modern Syntax: A concise way to write functions introduced in ES6.

Syntax:

let add = (a, b) => a + b;
console.log(add(5, 3));  // Outputs: 8

Benefits: Shorter syntax, especially useful for one-liners. Arrow functions also handle the this context differently, which can be useful in certain situations.

8. IIFE (Immediately Invoked Function Expressions)

Definition: A function that is executed right after it is defined.

Syntax:

(function() {
    console.log("This function runs immediately!");
})();

Use Cases: Useful for creating a private scope to avoid polluting the global namespace.

9. Higher-Order Functions

Definition: A function that takes another function as an argument or returns a function.

Example:

function multiplyBy(factor) {
    return function(x) {
        return x * factor;
    };
}
let double = multiplyBy(2);
console.log(double(5));  // Outputs: 10

Use Cases: Common in functional programming, for tasks like mapping, filtering, and reducing arrays.

10. Callback Functions

Definition: A function passed as an argument to another function and executed inside that function.

Example:

function greet(name, callback) {
    console.log("Hello, " + name);
    callback();
}
function afterGreeting() {
    console.log("Welcome to the platform!");
}
greet("Alice", afterGreeting);
// Outputs:
// Hello, Alice
// Welcome to the platform!

11. Closures

Definition: A closure is a function that remembers its lexical scope even when the function is executed outside that scope.

Example:

function outerFunction() {
    let outerVariable = "I'm outside!";
    return function innerFunction() {
        console.log(outerVariable);
    };
}
let myFunction = outerFunction();
myFunction();  // Outputs: I'm outside!

Use Cases: Closures are useful for data encapsulation and creating private variables.

12. Default Parameters

Definition: Assign default values to function parameters if no argument is provided.

Syntax:

function greet(name = "Guest") {
    console.log("Hello, " + name + "!");
}
greet();  // Outputs: Hello, Guest!

13. Rest Parameters

Definition: Collect all remaining parameters into an array.

Syntax:

function sum(...numbers) {
    return numbers.reduce((total, num) => total + num, 0);
}
console.log(sum(1, 2, 3, 4));  // Outputs: 10

14. Function Scope and Hoisting

Function Scope: Functions have access to their own scope, the outer scope (closure), and the global scope.

Hoisting: Function declarations are hoisted to the top of their scope, meaning they can be called before they are defined in the code.

Example of Hoisting:

console.log(add(2, 3));  // Outputs: 5
function add(a, b) {
    return a + b;
}
